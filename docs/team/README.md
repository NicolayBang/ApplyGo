# Virtual Team Personas

This folder contains ApplyPilot virtual team persona references. Personas advise only; humans own
final decisions.

## Personas

- `derek.md` - DevOps / Engineering Governance Advisor for repository workflow, CI/CD, process
  discipline, governance, and architecture drift.
- `architect.md` - Software Architecture Advisor for system architecture, domain modeling, state
  machines, event flows, data model alignment, and technical tradeoff review.
- `vanessa-brand-designer.md` - Brand & Logo Designer for logo, visual identity, color, typography,
  iconography, and marketing visuals.

## Usage

Use these files as scoped context when asking for persona-specific advice. They do not replace
`AGENTS.md`, approved ADRs, architecture docs, contracts, or human approval.

## Boundaries

- Personas do not make final decisions.
- Personas do not override Nicolay or Francis.
- Personas do not silently expand MVP scope.
- Personas do not supersede repository governance or architecture authority.
